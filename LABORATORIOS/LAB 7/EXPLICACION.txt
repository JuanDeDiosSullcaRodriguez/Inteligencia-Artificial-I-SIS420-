este documento resume un laboratorio sobre como trabajar en machine learning cuando faltan datos etiquetados el proyecto arranca usando datos inventados para probar un algoritmo que agrupa elementos acercandolos a centros calculados al azar el gran problema aqui es que el resultado depende mucho de donde caen esos centros iniciales y como en la vida real nadie te dice cuantos grupos existen de antemano se enseñan trucos visuales y matematicos como el metodo del codo para descubrir la cantidad ideal midiendo que tan bien encaja cada dato en su respectivo grupo
luego la practica pasa al mundo real usando fotografias de arquitectura las cuales se procesan para que la computadora las entienda mejor primero se aplica un enfoque no supervisado donde el sistema agrupa las fotos a ciegas y busca las imagenes mas representativas de cada conjunto despues entra el aprendizaje semi supervisado que consiste en etiquetar a mano solo esas fotos principales y entrenar el modelo con ellas logrando resultados muy buenos sin tener que clasificar todas las imagenes manualmente aunque a veces se intenta copiar esa etiqueta a todo el grupo corriendo el riesgo de generar confusiones
finalmente se explora el aprendizaje activo que es dejar que el algoritmo trabaje y nos avise con cuales fotos tiene mas dudas para que un humano intervenga y etiquete solo esas imagenes dificiles al corregir esos casos puntuales el rendimiento mejora muchisimo esta practica demuestra como lidiar con bases de datos imperfectas usando librerias de codigo conocidas combinando varias estrategias para no gastar tiempo etiquetando elementos innecesarios lo cual resulta ser una habilidad vital para proyectos reales en hospitales o empresas de tecnologia
